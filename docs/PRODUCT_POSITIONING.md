## Why "Website System" and Not Just "Template"

A template is a visual starting point. You get colors, fonts, and a layout — then figure out the rest yourself.

A **website system** includes the invisible architecture that actually converts visitors:

### 1. Patient Journey Architecture
Every page has a defined role in the conversion funnel:
- **Homepage** → Brand positioning + emotional connection
- **Protocol pages** → Methodology proof + differentiation
- **Technology pages** → Scientific credibility + authority
- **Treatment catalog** → Service discovery + filtering
- **Academy pages** → Education + trust deepening
- **Consultation page** → Conversion + lead capture

### 2. Conversion Logic Built In
- Hero sections with dual CTAs (high-intent and browsing paths)
- Trust strips with credential and experience indicators
- Treatment cards linking to consultation
- Protocol system that builds confidence in methodology
- Testimonial sections for social proof
- Persistent consultation CTAs throughout

### 3. Content Strategy
- Treatment catalog with real-time search and category filtering
- Technology modals for quick education without leaving the page
- Protocol detail pages that explain the "why" behind treatments
- Academy section for patient education and authority

### 4. CMS-Ready Architecture
- 7 collection schemas defined with required/optional fields
- Example entries for every collection
- Webflow binding recommendations
- Dynamic page route suggestions

### 5. Documentation Package
- README with full customization guide
- Setup Guide with technical details
- Onboarding Guide for 48-hour launch
- Product Structure explaining conversion logic
- CMS Structure with complete schemas
- Publishing Checklist for marketplace readiness
